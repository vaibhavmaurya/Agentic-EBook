# Ebook Platform — Chapter Style Guide

> This file is loaded into all writing and editorial agent prompts.
> Update this file to globally adjust content standards across all topics.

## Tone and Voice

- Authoritative but accessible — write for an informed professional audience
- Avoid filler phrases: "In conclusion", "It is worth noting", "As mentioned above"
- Use active voice wherever possible
- Define acronyms on first use within each section

## Structure Requirements

Every chapter must include:
1. A short introductory paragraph (2–3 sentences) summarizing the topic
2. Sections organized by logical subtopics (use H2 headings)
3. At least one Mermaid sequence diagram illustrating the main concept
4. At least one Mermaid block diagram for architecture or data flow (if applicable)
5. A "Key Takeaways" subsection at the end of each major section
6. A "References" section listing all cited sources with URLs

## Diagram Standards

Use Mermaid syntax. Sequence diagrams preferred for:
- Process flows, API calls, agent interactions, request/response patterns

Block diagrams preferred for:
- System architecture, data flow, component relationships

Keep diagrams simple — maximum 8 nodes per diagram.

## Citations

- Cite sources inline using numbered references: `[1]`, `[2]`, etc.
- List full citations in the References section: `[1] Title — URL — Accessed YYYY-MM`
- Prefer primary sources (official documentation, research papers, vendor guides)
- Do not cite sources older than 3 years unless foundational/seminal

## Prohibited Content

- Do not include speculative forward-looking predictions as facts
- Do not reproduce copyrighted content verbatim
- Do not include pricing information (changes frequently)
- Do not make comparisons that could be interpreted as vendor endorsements
